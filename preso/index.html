Introduction

Het probleem

  • Om te laten zien hoe mijn code werkt moet ik switchen van de omgeving
  • Sublime Text whatever -> Browser (misschien zelfs met page refresh)
  • Terug naar Sublime Text whatever
  • Wat doet bepaalde stuk code?

Inspiratie

  • Bret Vicor met zijn Inventing on Principle talk en Lernable Programming essay
  • De presentatie van Lea Verou tijdens CSS Day afgelopen juni in Amsterdam
Immediate-update is merely a prerequisite for doing anything interesting
Bret Victor

Hoe zou het moeten

De principes (de meeste uit Inventing on Principle en Learnable Programming)

  • Show, don't tell
  • Follow the flow (bijvoorbeeld, timelines, play/pause)
  • See the state (wat gebeurt op dat punt)
  • Create by reacting

Concrete oplossing

De tool

  • Code zou visueel inzichtelijk gemaakt worden (ga uit van iemand die jouw code niet kent)
  • Door immediate connection krijg het publiek veel meer grip op je code
  • Highlight een stuk code en maak het mogelijk het te wijzigen en meteen de wijzigingen zien
  • Moet als een extentie werken op Dev Tools, meer nadruk op code en wat het doet

Klinkt als veel, he?

  • Je zou zo veel opnieuw moeten opzetten!

  • ...

Nou, waarom gebruiken we die bestaande tools niet voor bedenktijdjes?

Het idee is een laag bovenop te creeren, net als Dev Tools, zodat we geen nieuwe opzet hoeven te maken
