Nou, waarom gebruiken we die bestaande tools niet voor bedenktijdjes?

Het idee is een laag bovenop te creeren, net als Dev Tools, zodat we geen nieuwe opzet hoeven te maken

Scope

  • Een prototype voor het testen van een feature
  • Bijvoorbeeld css3 animations en transitions
  • Niet voor backend, nee
  • Omdat het debuggen van css3 animations in dev tools onmogelijk is
  • Het liefst geintegreerd met dev tools (nader onderzoek nodig)

Haalbaar?

  • Javascript opzet van editing omgeving
  • Keuze voor platform
  • Het wordt een prototype!
  • We gaan (nog) niet integreren met Dev Tools

Vet?

  • Niet alleen voor bedenktijdjes, maar ook voor product demo's!
  • Een eigen Q product

De toekomst

  • Een Chrome Extension
  • Mogelijk bestaande code (binnen een bestaand project) te testen
  • Integreren met source maps voor het eventueel opslaan van de wijzigingen
  • ...
